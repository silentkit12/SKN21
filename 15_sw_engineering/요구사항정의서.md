# [요구사항 정의서] 일반인 대상 쉬운 법률 Q&A 챗봇

## 1. 프로젝트 개요
* **프로젝트명:** 일반인 대상 쉬운 법률 Q&A 챗봇 (가칭: 법률 가이드봇)
* **목적:** 복잡한 법령 용어를 일상 언어로 풀이하여 사용자 상황에 맞는 법률 정보를 제공하고 법률 서비스 문턱을 낮춤.
* **핵심 기술:** 
    * **Framework:** Django, Bootstrap
    * **AI:** RAG, LLM API - GPT4.0
    * **Database:** Vector Database, Qdrant, Docker
    * **Infrastructure:** AWS EC2

---

## 2. 요구사항 명세 (Requirements Specification)

### 2.1 기능적 요구사항 (Functional Requirements)

| ID | 요구사항명 | 상세 설명 | 비고 |
| :--- | :--- | :--- | :--- |
| **F-01** | **사용자 인증** | Google OAuth 및 일반 이메일(ID/PW) 가입 기능을 모두 제공함. | Google OAuth 2.0 |
| **F-02** | **채팅 UI** | 실시간 질의응답이 가능한 웹 기반 채팅 인터페이스(UI/UX) 제공. | 반응형 웹 디자인 |
| **F-03** | **법률 질의응답** | 일상어 질문을 분석하여 관련 법령 근거와 함께 쉬운 해설 제공. | 3차 프로젝트 RAG 모듈 연동 |
| **F-04** | **대화 이력 저장** | 이메일 주소를 고유 식별자로 사용하여 대화 이력을 통합 관리/로드함. | 유저별 세션 관리 |
| **F-05** | **근거 법령 제시** | 답변의 신뢰성을 위해 참조한 법령 조문을 명확히 명시. | 출처 표시 기능 |

### 2.2 비기능적 요구사항 (Non-functional Requirements)

| ID | 구분 | 상세 설명 |
| :--- | :--- | :--- |
| **N-01** | **신뢰성** | 환각 현상 방지를 위해 검색된 법령 데이터 범위 내에서 답변을 생성하도록 프롬프트 제어. |
| **N-02** | **가용성** | AWS EC2 인프라를 활용하여 안정적인 서비스 접속 환경 유지. |
| **N-03** | **성능** | 질문 입력 후 답변 생성 시작까지 최대 5초 이내 수행 (Streaming 처리 권장). |

## 3. 시스템 아키텍처 (System Architecture)

1.  **Client (User):** 웹 브라우저를 통한 질문 입력 및 히스토리 조회.
2.  **Django Server:** 웹 서비스 로직, 유저 인증(Google), 대화 이력 관리.
3.  **Vector DB:** 구조화된 법령 조문 데이터를 저장하고 유사도 기반 검색 수행.
4.  **LLM API:** 검색된 조문을 참고하여 사용자 맞춤형 쉬운 답변 생성.
5.  **Deployment:** AWS EC2를 통한 서버 호스팅.

---

## 4. 제약 사항 (Constraints)
* **데이터 활용:** 법령 데이터 구조화 및 RAG 엔진은 3차 프로젝트 성과물을 기반으로 함.
* **법적 고지:** "본 서비스의 답변은 법적 효력이 없으며, 전문가 상담을 대체할 수 없다"는 면책 고지(Disclaimer)를 UI에 상시 노출함.
